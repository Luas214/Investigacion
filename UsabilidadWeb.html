Que es la usabilidad Web

La usabilidad web es un concepto que siempre hay que tener en cuenta a la hora de crear una página web. La usabilidad web se refiere al grado de facilidad para acceder y navegar por una página de forma intuitiva, sencilla y rápida

Importancia de la usabilidad Web

Al fin y al cabo la usabilidad de una página web es fundamental si queremos que un usuario permanezca en esta, para esto esta debe de ser de un fácil uso, agradable a la vista para que esta no genere confusiones, que sea lo mas sencillo posible, todo esto nos va a permitir tener una pagina en condiciones

Principios de la Usabilidad Web

  • Visibilidad del estado del sistema

    Implica que el sitio web debe mantener a los usuarios informados sobre lo que está ocurriendo en todo momento. Esto se logra mediante retroalimentación clara y rápida

    Ejemplo: A la hora de querer realizar una compra, hacer que la pagina deje claro en qué proceso nos encontramos


  • Relación entre el sistema y el mundo real

    Sugiere que el sistema debe utilizar lenguaje, iconos, y conceptos familiares para los usuarios, con el fin de que el sitio sea más comprensible e intuitivo

    Ejemplo: A la hora de borrar un elemento, se usa la papelera como símbolo de borrar


  • Control y libertad del usuario

    Indica que los usuarios deben tener el control sobre el sistema y la libertad de revertir o deshacer acciones si es necesario

    Ejemplo: El borrar artículos del carro de compra, o el botón de deshacer


  • Consistencia y estándares

    La pagina debe seguir los convenios establecidos para ciertos iconos, además señala que los usuarios deben poder interactuar con el sistema de manera consistente. Los mismos elementos, términos y acciones deben comportarse de manera uniforme a lo largo del sitio

    Ejemplo: Los botones verdes los asociamos a aceptar una cosa, y los botones rojos a cancelar


  • Prevención de errores

    Los sistemas deben estar diseñados para minimizar la posibilidad de que los usuarios cometan errores. Esto implica ofrecer advertencias claras, restricciones en entradas de datos cuando sea posible, etc

    Ejemplo: Comprobación de campos de formularios en tiempo real


  • Reconocer antes que recordar

    Sugiere que los sistemas deben facilitar que los usuarios reconozcan información en lugar de tener que recordarla. Los elementos de la interfaz deben ser visibles y familiares, y las acciones comunes deben estar a la vista

    Ejemplo: Un sistema recuerda las búsquedas recientes o las opciones más usadas y las sugiere automáticamente


  • Flexibilidad y eficiencia de uso

    Establece que los sistemas deben ser flexibles y adaptarse tanto a usuarios novatos como a usuarios avanzados

    Ejemplo: En un programa de edición de texto, los usuarios novatos pueden utilizar menús y botones para dar formato al texto, mientras que los usuarios avanzados pueden utilizar combinaciones de teclas (como Ctrl+B para negrita)


  • Diseño estético y minimalista

    Indica que el diseño de la interfaz debe ser simple y libre de elementos innecesarios, permitiendo que los usuarios se concentren en las tareas más importantes

    Ejemplo: Elimina todo lo que consideres innecesario y que no aporta nada a lo que quieres decir.


  • Ayudar a los usuarios a reconocer, diagnosticar y corregir los errores

    enfatiza la importancia de proporcionar a los usuarios herramientas y mensajes que les ayuden a identificar y resolver problemas cuando ocurren errores

    Ejemplo: El error 404 es un error que una gran cantidad de personas saben identificar


  • Ayuda y documentación

    Establece que los sistemas deben ofrecer ayuda y documentación accesibles para asistir a los usuarios cuando lo necesiten. Aunque es preferible que el sistema sea intuitivo y no requiera asistencia

    Ejemplo: Debería de haber algún tipo de manual de funcionamiento para los usuarios
